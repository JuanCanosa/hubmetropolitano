00 — O diagnóstico

Por que 1 Listing Type não funciona para Serviços

Serviços é a categoria mais ampla e diversa do Hub. Um único template trata igual uma agência de marketing digital, um contador, um eletricista e um pet sitter — profissionais com necessidades, campos, schemas e SEO radicalmente diferentes.

❌ Antes — Um Listing Type genérico
🚫 Agência de marketing e encanador no mesmo template
🚫 Schema ProfessionalService genérico — SEO fraco
🚫 Sem forma de listar serviços individuais com preço
🚫 Técnico não tem campo para urgência e área de cobertura
🚫 Contador não tem campo para CRC e registro profissional
🚫 Agência não tem campo para cases e portfólio de projetos
🚫 URL /servicos/ sem especificidade — ranking diluído
🚫 ~1.200 pesquisas/mês capturadas no total
✅ Depois — 3 Listing Types especializados
✅ Cada tipo com campos e UX adequados ao perfil
✅ Schemas distintos: Organization, FinancialService, HomeAndConstruction
✅ Sistema "O que Fazemos" — cards individuais por serviço
✅ Urgência + área de cobertura para técnicos
✅ CRC, CRECI, CREA por tipo de profissional
✅ Cases + portfólio + resultados para agências
✅ 3 URLs semânticas especializadas
✅ ~14.800 pesquisas/mês potenciais — 12× mais

02 — A Feature Principal

✨ Sistema "O que Fazemos" — Cards de Serviços por Profissional

Esta é a feature que separa o Hub de qualquer diretório no Brasil. Cada profissional cria até 8 cards individuais — com ícone, nome, descrição e preço opcional. A agência lista "Tráfego Pago, Sites, Automação". O eletricista lista "Instalações, Quadros, Emergências 24h". O visitante vê imediatamente o que pode contratar — e decide no perfil, sem precisar ligar para perguntar.

🔧 Implementação do Repeater "O que Fazemos": No MyListing, use o campo tipo Repeater no formulário. Cada repetição tem 4 subcampos: servico-icone (emoji, texto livre), servico-nome (obrigatório), servico-descricao (textarea, máx. 120 char) e servico-preco (opcional). No Single Page, use um bloco custom que percorre os itens do Repeater e renderiza os cards em grid 2 ou 3 colunas. Se o Repeater não estiver disponível, a alternativa é um campo Textarea com formato estruturado + shortcode PHP para renderizar.

07 — Schema

Schema com hasOfferCatalog — O diferencial de SEO

O schema hasOfferCatalog é o que conecta o perfil da agência com cada serviço individual do "O que Fazemos". O Google processa cada serviço como uma Offer distinta — aumentando a probabilidade de ranquear para "tráfego pago barra da tijuca" mesmo que o título do perfil seja o nome da agência.

Agências & Digitais · @type ProfessionalService + hasOfferCatalog 
{
  "@context": "https://schema.org",
  "@type": ["ProfessionalService", "LocalBusiness"],
  "name": "[[title]]",
  "description": "[[description]]",
  "url": "[[:url]]",
  "telephone": "[[phone]]",
  "email": "[[email]]",
  "sameAs": "[[links]]",
  "hasOfferCatalog": {          // Cada item = um card do "O que Fazemos"
    "@type": "OfferCatalog",
    "name": "Serviços Oferecidos",
    "itemListElement": [
      {
        "@type": "Offer",
        "itemOffered": {
          "@type": "Service",
          "name": "[[servico-nome-1]]",
          "description": "[[servico-descricao-1]]"
        },
        "price": "[[servico-preco-1]]",
        "priceCurrency": "BRL"
      }
      // Repetir para servico-nome-2, servico-nome-3... até 8
    ]
  },
  "potentialAction": {
    "@type": "QuoteAction",
    "target": "[[proposta-link]]",
    "name": "Solicitar Proposta"
  },
  "address": {
    "@type": "PostalAddress",
    "addressLocality": "Rio de Janeiro",
    "addressRegion": "RJ",
    "addressCountry": "BR"
  },
  "aggregateRating": {
    "@type": "AggregateRating",
    "ratingValue": "[[:reviews-average]]",
    "reviewCount": "[[:reviews-count]]"
  }
}
🏆 Por que o hasOfferCatalog muda o SEO: Quando o Google processa este schema, entende que a Orbital Agency oferece especificamente "Tráfego Pago", "Criação de Sites" e "Automação" como serviços distintos. A probabilidade do listing aparecer em buscas por cada serviço individualmente aumenta significativamente — não só pela categoria genérica "Agência Digital". É a diferença entre ranquear para 1 keyword vs 8 keywords com um único perfil.
08 — Implementação

Executar nesta ordem — 3 Listing Types do zero

💻 Agências & Digitais — ~90min
1Criar 3 taxonomias: modalidade-digital, publico-digital, faixa-digital
2Criar 6 categorias com ícones e cores: marketing digital, dev web, design, tráfego pago, automação, educação
3Criar Listing Type · Permalink: digital
4Configurar 5 Grupos de Campos incluindo o Repeater "O que Fazemos"
5Configurar Reviews (5 critérios de digital)
6Single Page com bloco de cards do Repeater em destaque
7Schema ProfessionalService + hasOfferCatalog
8Explore Page · Rank Math configurado
🏢 Profissionais B2B — ~75min
1Criar 3 taxonomias: publico-b2b, modalidade-b2b, faixa-honorarios
2Criar 5 categorias: contabilidade, imobiliária, seguros, consultoria, RH
3Criar Listing Type · Permalink: servicos-profissionais
4Campos incluindo registro-profissional e setores-especialidade
5Repeater "O que Fazemos" com 4 subcampos
6Reviews (5 critérios B2B)
7Schema FinancialService / ProfessionalService
8Explore Page + Rank Math
🔧 Técnicos & Casa — ~75min
1Criar 3 taxonomias: atende-urgencia, disponibilidade-tec, area-cobertura (como taxonomy)
2Criar 6 categorias: reforma, elétrica, ar-condicionado, pet care, transporte, limpeza
3Criar Listing Type · Permalink: servicos-tecnicos
4Campos incluindo prazo-atendimento e garantia
5Repeater "O que Fazemos" com preço por serviço
6Reviews (5 critérios técnicos)
7Schema HomeAndConstructionBusiness
8Badge de urgência visível no Card e Single Page
⚠️ O que fazer com o Listing Type "Serviços" anterior: Se já houver listings cadastrados no tipo antigo, migre-os para o tipo correto antes de despublicar o tipo antigo. Exporte os dados via plugin de export do MyListing, reatribua ao novo CPT, e configure um redirect 301 de /servicos/[slug] para o novo URL. Nunca delete o tipo antigo sem redirect — você perde qualquer link ou indexação já existente.
